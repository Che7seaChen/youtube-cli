# youtube-cli 0.2.0

`youtube-cli` 0.2.0 adds risk-control modes and a doctor command to help reduce trigger rates and diagnose environment/auth issues.

`youtube-cli` 0.2.0 新增风控模式与 doctor 诊断命令，用于降低触发率并快速定位环境/认证问题。

## Added / 新增

- `--mode safe|balanced|fast` for risk-control pacing
- `youtube doctor` for environment/auth diagnostics and recent error metrics
- local metrics tracking for command success/failure (no URL logging)

- 新增 `--mode safe|balanced|fast` 风控节奏模式
- 新增 `youtube doctor` 诊断环境/认证与最近错误指标
- 新增本地指标记录（不记录 URL）

## Changed / 变更

- applied request sleep intervals to read flows and task-level jitter for batch downloads
- added retry/backoff for write APIs and explicit rate-limited handling on write calls
- status output now surfaces risk-control configuration

- 读链路加入请求间隔、批量任务加入抖动
- 写接口增加重试/退避与限流提示
- status 输出包含风控配置

## Representative Commands / 代表性命令

```bash
youtube --no-check-certificate doctor
youtube --mode safe video "https://www.youtube.com/watch?v=VIDEO_ID"
youtube --mode safe download "https://www.youtube.com/watch?v=VIDEO_ID" --quality 360p
```

## Requirements / 依赖

- Python 3.11+
- `ffmpeg`
- `yt-dlp >= 2026.3.3`
- optional / 可选: `aria2c`
